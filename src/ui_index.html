WRITE THIS DOWN — only chance

If the Keychain item is lost (Mac wipe, biometric reset, OS reinstall), this 24-word phrase is the ONLY way to recover the wallet. Store it on paper, in a safe.

Send XOR

Submitting will trigger a Touch ID prompt, sign with the wrapped seed, and POST the signed transaction to https://minamoto.sora.org/transaction.

Shield XOR (privacy)

Convert public XOR into a confidential note. The chain records only a 32-byte Poseidon commitment and an opaque payload — no observer can see the amount or owner. Phase-1 limitation: we ship the encryption-payload as zeros so this wallet must keep the local note record to spend the note later. Lose the local file → the XOR stays on chain but is not spendable.

My confidential payment address (v3)

A shareable URI anyone can use to send you confidential XOR. The URI carries your shielded owner_tag + diversifier — without your seed, nobody can spend the notes that target it. Generating it requires Touch ID (we derive the material from the wallet seed) but you only need to do it once and can publish it on Twitter / a website / a QR code.

Pay confidential address

Shield XOR directly to a third-party iroha:confidential:v3:… URI. The on-chain commitment will be owned by THEM, not us — so we don't keep a local note copy. Phase-1 limitation: the recipient cannot auto-discover the note from the chain (we ship the encryption payload as zeros). After a successful pay, share the returned rho with them out-of-band (DM, Signal) so they can register the note locally and spend it later.

Local shielded notes

Each Shield you create persists here. They are required to spend the note later (Phase 2). Status spendable = the note has not been used as input of a future Unshield/ZkTransfer.
